Introduction

The four important components of this visualization are:
  • index.html
  • scroller.js
  • section.js
  • style.css


index.html contains the website metadata and text content.

section.js handles data input and draws the visualization using D3.js.

style.css provides additional styling details.

scroller.js adds the scrolling-updating component to the website.

Testing of your web page may be done through a local web server, implemented through Python. After setting your working directory to where the HTML is located, use the following:

python3 -m http.server

For sharing to broader audiences, code may be commited to github and made accessible through github.io.

index.html

This file can be broken into two major sections: head and body. For HTML, the head and body can be distinguished through the tags < head > < / head > or < body > < / body > . Understanding how tags are used is essential to writing an HTML web page.

The head contains the web page title and meta data. The title appears as text within a tab of a web browser. Meta data provides additional information about the page.

In order to use D3.js, it has to be called on, as seen in line 6 of this file. You may click on the header to access the HTML file, available through Github. Since the D3.js code is available online, it doesn't have to downloaded or installed locally to function as long as it is properly referenced.

The text you've been reading through is contained with the body section. Within the body, I've used three HTML tags for Header 1, Header 2, and Paragraph. Additionally, text is particularly located in 'step' sections that incorporate the interactive scrolling, explained below.

sections.js

The network graph is visualized within the sections.js file. This occurs when sections.js is called at the end of index.html. I'll cover the flow of code to interpret how separate files are now coordinating the visualize the graph on a single page.

First, there is a < div > section with the ID 'viz', located below the 'step' sections. Within sections.js, this 'viz' section is assigned an Scalable Vector Graphic (SVG) with width, height, and margin parameters. This SVG is the canvas, which the network graph is drawn on.

The graph data, as a JSON file, is imported. The graph has keys for nodes and links. Each node has a unique ID and will be drawn as a circle on the graph. Each link has a source and target that connects nodes together. The layout of the nodes is then determined by a force simulation. The nodes are centered around the SVG and ultimately settle on a location, based on some repulsion/attraction paramters.

The final position of the nodes occurs iteratively from when all nodes are initially drawn on the top left corner. The iterations occur quick enough that it's apparently instantaneous.






At the bottom of sections.js, there are functions for actions on scrolling. At minimum, I've maintained the fade in/out of text as the user navigates 'step' sections, compared to Cuthbert Chow's original code. This occurs as the scroller maintains an index of the 'step' sections and tracks user placement on the page. Inactive indices are faded out to maintain focus on relevant text.

Within sections.js, I would also plan to code animations, tool tips, etc. for the visualization on the right. However, given the intended challenge of creating a custom visualization, I have not yet incorporated animating the network graph.

I am settling for a single network graph visualization for this assignment, but will continue to build upon this for the final visualization.
